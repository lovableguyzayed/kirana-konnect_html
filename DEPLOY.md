# Deploying Kirana Konnect on Render

This app deploys entirely on **Render** — both the web app and the PostgreSQL
database. You do **not** need Supabase, Neon, or any external service, and you
do **not** need to set any environment variables by hand. The `render.yaml`
Blueprint provisions everything.

## What gets deployed

`render.yaml` defines two resources:

1. **Web service** (`kirana-konnect`) — runs the database-backed `app.py` with
   Gunicorn.
2. **PostgreSQL database** (`kirana-konnect-db`) — Render's free managed Postgres.

Render automatically injects the database connection into the app, so the app
connects on first boot, creates all tables, and seeds sample data.

## Steps

1. Make sure `render.yaml`, `requirements.txt`, and `app.py` are on the branch
   Render will deploy (e.g. `main`).
2. Go to <https://dashboard.render.com> and sign in (a free account, GitHub
   login works).
3. Click **New +** -> **Blueprint**.
4. Connect GitHub and select the `kirana-konnect_html` repository.
5. Render reads `render.yaml` and shows the plan: one web service + one free
   Postgres database. Click **Apply**.
6. Wait a few minutes for the build. When it finishes you get a public URL like
   `https://kirana-konnect.onrender.com`.

That's it. No connection strings to copy, no secrets to paste.

## Environment variables (all handled automatically by render.yaml)

| Variable | How it is set on Render | Purpose |
|----------|-------------------------|---------|
| `DATABASE_URL` | Pulled from the Render Postgres database (`fromDatabase`) | Database connection. `app.py` normalizes `postgres://` to `postgresql://`. |
| `SESSION_SECRET` | Auto-generated by Render (`generateValue: true`) | Flask session signing key. |
| `PORT` | Provided by Render at runtime | Port Gunicorn binds to (`--bind 0.0.0.0:$PORT`). |
| `PYTHON_VERSION` | Set to `3.11.9` in `render.yaml` | Python version for the build. |

You only ever touch these manually if you later switch to an external database.

## Free-tier notes

- The free **web service sleeps after ~15 minutes** of inactivity; the first
  request after idle takes ~30-60 seconds to wake it.
- The free **Postgres database expires 30 days** after creation. Render emails
  you beforehand; upgrade the database or create a new free one to continue.

## Running locally (optional)

```bash
pip install -r requirements.txt
cp .env.example .env          # then fill in DATABASE_URL + SESSION_SECRET
gunicorn app:app --bind 0.0.0.0:5000
```

`app.py` loads `.env` automatically via python-dotenv.
